# Personal Intent (for programmers)

## Modes (pick 1–2)
- ship: deliver a working outcome quickly
- learn: understand deeply, accept slower progress
- explore: try options, prototype, compare
- refactor: improve design, reduce complexity
- stabilize: reduce bugs/risks, add tests/observability
- mentor: make it easy for others (docs, clarity, onboarding)
- document: improve future recall and handoffs
- de-risk: minimize safety/security/compliance hazards

## Common values (examples)
- correctness, simplicity, clarity, speed, reliability
- maintainability, auditability, security, safety
- autonomy, confidence, craft, calm, focus

## Typical tradeoff axes
- speed vs correctness
- flexibility vs simplicity
- automation vs control
- novelty vs reliability
- local optimization vs long-term architecture

## Anti-goals (examples)
- “no rewrites”
- “no brittle magic”
- “no hidden state”
- “no risky automation”
- “no deep rabbit holes today”

## Definition of done patterns
- “works on my machine + CI”
- “documented + examples”
- “tests cover happy path + edge”
- “no known critical risks”
